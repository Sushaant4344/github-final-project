Contributing to Simple Interest Calculator
All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.
We are excited that you are interested in contributing to this project. By participating in this project, you agree to abide by our Code of Conduct.
How to Contribute
Reporting Bugs
If you find a bug, please open an issue on GitHub. When reporting a bug, please include:
A clear description of the issue.
Steps to reproduce the error.
Your environment (e.g., OS, browser version).
Suggesting Enhancements
If you have ideas for new features or improvements, feel free to open an issue labeled "enhancement" to discuss the proposal with the maintainers.
Pull Requests
If you would like to submit a code change:
Fork the repository.
Create a branch for your feature or fix (git checkout -b feature/amazing-feature).
Make your changes, ensuring the code is clean and commented where necessary.
Test your changes to ensure they work as expected.
Commit your changes with a descriptive message.
Push to your branch (git push origin feature/amazing-feature).
Open a Pull Request against the main branch.
Style Guidelines
Keep code simple and readable.
Ensure all new features include relevant documentation updates.
Follow the existing indentation and naming conventions used in the project.
Attribution
This guide is based on standard open-source contribution practices to ensure a positive and productive experience for all contributors.
